# Annotation Flowchart Builder

アノテーション用のタグセット、判定フロー、オーバーライドルールを作成・編集し、フローチャートと実行画面として使えるブラウザアプリです。

公開版:

https://sakana0106hry.github.io/annotation_flowchart/

## このREADMEの読み方

- アプリを使うだけの場合は、`ユーザー向け` を読んでください。
- GitHubで管理したり、コードを直したり、公開版を更新する場合は、`開発者向け` を読んでください。
- 公開版には非公開タグセットや非公開ルールは含まれていません。必要なタグセットJSONを別途読み込んで使います。

## ユーザー向け

### できること

- 複数のタグセットを作成・切り替え
- タグ、タグ説明、系統、判定フロー、分岐、オーバーライドルールを画面から編集
- 判定をドラッグアンドドロップで並び替え
- 編集内容をブラウザに自動保存
- フローチャートをSVG / PNG / Mermaidで出力
- ~アノテーション結果をCSVで出力~
- 選択中のタグセット、または全タグセットをJSONで読み書き

### 基本の使い方

1. 公開版URLを開く
2. 上部の `メニュー` を開く
3. `タグセット` で新規作成、複製、切り替えを行う
4. `基本情報`、`タグ管理`、`オーバーライド`、左側の `ルール編集` で内容を編集する
5. 必要に応じて `入出力` からJSONを書き出す

### データの保存

画面上で編集した内容は、使用中のブラウザの `localStorage` に自動保存されます。

- 同じPC・同じブラウザで再度開くと、編集内容が残ります。
- 別PCや別ブラウザには自動では共有されません。
- 共有やバックアップが必要な場合は、`メニュー > 入出力 > 全タグセットを書き出す` でJSONを書き出してください。
- JSONを受け取った人は、`メニュー > 入出力 > JSONを読み込む` から読み込めます。

### JSONの使い分け

- `このタグセットを書き出す`: いま選んでいるタグセットだけをJSONにします。
- `全タグセットを書き出す`: アプリ内の全タグセットをまとめてJSONにします。バックアップ向きです。
- `JSONを読み込む`: 書き出したJSONをアプリに読み込みます。

## 開発者向け

### セットアップ

```bash
npm install
npm run dev
```

Windows PowerShell の実行ポリシーで `npm` が止まる場合:

```bash
npm.cmd install
npm.cmd run dev
```

### 主なファイル

- `src/App.tsx`: 画面全体、編集UI、入出力処理
- `src/logic.ts`: アノテーション結果、CSV、Mermaid、JSON出力のロジック
- `src/storage.ts`: `localStorage` の読み書き、タグセットの正規化
- `src/rules.ts`: 公開用の空テンプレート
- `src/types.ts`: タグセット、判定ステップ、分岐などの型定義
- `src/styles.css`: 画面デザイン

### ビルド

通常ビルド:

```bash
npm run build
```

GitHub Pages向けビルド:

```bash
npm run build:pages
```

### デプロイ

`main` ブランチにpushすると、GitHub Actionsで自動ビルドされ、GitHub Pagesへ公開されます。

GitHub Pages:

https://sakana0106hry.github.io/annotation_flowchart/

### 非公開データの扱い

タグセットやアノテーションルールが非公開情報の場合は、アプリ本体とは分離してください。

- 非公開のJSONやルール定義は `private/` に置くとGitの対象外になります。
- `src/rules.ts` は公開用の空テンプレートです。非公開ルールを直接書き込まないでください。
- `node_modules/`、`dist/`、`private/`、ログファイルは `.gitignore` で除外しています。
- GitHubにpushする前に、非公開タグ名や非公開ルールが混ざっていないことを確認してください。

### 公開前チェック

```bash
git status --short
npm run build:pages
```

非公開語句が混ざっていないか確認する例:

```bash
git grep -n "非公開タグ名"
```

### JSON互換性

既存のタグセットJSONを使えるようにするため、保存形式を変える変更は慎重に扱ってください。UI文言や並び替え機能を変更しても、基本的には既存JSONを読み込める状態を保つ方針です。
